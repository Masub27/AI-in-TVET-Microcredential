# Microcredential Quality Checklist

## Purpose

This checklist is used to review the quality of the AI-in-TVET Microcredential before submission or publication.

It helps check whether the micro-credential is clear, complete, assessed, and suitable for online and offline learning.

## Basic Information Checklist

- [ ] The micro-credential has a clear title.
- [ ] The project overview is included.
- [ ] The target group is clearly described.
- [ ] The aim of the micro-credential is explained.
- [ ] The estimated workload is included.
- [ ] The mode of participation is explained.
- [ ] The tools are listed.

## Learning Outcomes Checklist

- [ ] Learning outcomes are included.
- [ ] Learning outcomes are clear and understandable.
- [ ] Learning outcomes describe what learners can do.
- [ ] Learning outcomes are connected to the course modules.
- [ ] Learning outcomes are connected to assessment tasks.

## Content Checklist

- [ ] Course overview is complete.
- [ ] Micro-credential definition is included.
- [ ] EU standard elements are included.
- [ ] All modules are included.
- [ ] Module content is clear and organized.
- [ ] Materials are suitable for the target group.
- [ ] Examples are practical and relevant.

## Assessment Checklist

- [ ] Assessment strategy is included.
- [ ] Tasks are connected to learning outcomes.
- [ ] Assessment criteria are clear.
- [ ] Final project requirements are explained.
- [ ] Passing requirement is included.
- [ ] Evidence submission is explained.
- [ ] Assessment rubric is included.

## EU Standard Elements Checklist

- [ ] Learner identification is included in the certificate or badge.
- [ ] Title of micro-credential is included.
- [ ] Country or region is included.
- [ ] Awarding body is included.
- [ ] Date of issuing is included.
- [ ] Learning outcomes are included.
- [ ] Workload information is included.
- [ ] Level of learning experience is included.
- [ ] Type of assessment is included.
- [ ] Form of participation is included.
- [ ] Quality assurance information is included.

## Online Learning Checklist

- [ ] GitHub repository is organized.
- [ ] README.md explains the project clearly.
- [ ] Folders and files have clear names.
- [ ] Online links are working.
- [ ] GitHub Pages or LiaScript can be used if required.
- [ ] Learners can access materials online.

## Offline Learning Checklist

- [ ] Project folder can be downloaded.
- [ ] Offline folder structure is complete.
- [ ] Files can be opened without internet.
- [ ] Required images, audio, or videos are stored in assets.
- [ ] No important files are missing.
- [ ] Offline ZIP package can be created.

## AI Use Checklist

- [ ] AI-supported content is reviewed.
- [ ] AI-generated content is improved manually.
- [ ] AI use is mentioned in the reflection.
- [ ] Content is accurate and relevant.
- [ ] Content is suitable for the target group.
- [ ] Human review is visible.

## BMAD Method Checklist

- [ ] Build step is visible.
- [ ] Manage step is visible.
- [ ] Assess step is visible.
- [ ] Deliver step is visible.
- [ ] BMAD Method supports the course structure.
- [ ] BMAD Method is connected to the final project.

## Certificate or Badge Checklist

- [ ] Certificate template is included.
- [ ] Learner name field is included.
- [ ] Micro-credential title is included.
- [ ] Awarding body is included.
- [ ] Date field is included.
- [ ] Learning outcomes are included.
- [ ] Assessment result is included.
- [ ] Signature or reviewer field is included.

## Final Review Checklist

- [ ] All files are saved.
- [ ] Folder structure is complete.
- [ ] README.md is complete.
- [ ] Documentation files are complete.
- [ ] Module files are complete.
- [ ] Task files are complete.
- [ ] Templates are complete.
- [ ] Checklists are complete.
- [ ] Project is ready for GitHub.
- [ ] Project is ready for offline use.
