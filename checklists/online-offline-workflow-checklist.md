# Online and Offline Workflow Checklist

## Purpose

This checklist is used to review whether the AI-in-TVET Microcredential can work in both online and offline formats.

The project should be usable through GitHub, GitHub Pages, or LiaScript for online learning, and also through a downloaded folder or ZIP file for offline learning.

## Online Workflow Checklist

- [ ] GitHub repository is created.
- [ ] Project files are uploaded to GitHub.
- [ ] README.md explains the project clearly.
- [ ] Folder structure is visible and organized.
- [ ] Course overview is included.
- [ ] Learning outcomes are included.
- [ ] Module files are included.
- [ ] Task files are included.
- [ ] Assessment strategy is included.
- [ ] Templates are included.
- [ ] Checklists are included.
- [ ] Links are working.
- [ ] Learners can access the repository online.
- [ ] GitHub Pages or LiaScript can be used if required.

## Offline Workflow Checklist

- [ ] Project folder can be downloaded.
- [ ] Offline folder contains all required files.
- [ ] README.md can be opened without internet.
- [ ] Markdown files can be opened in VS Code or another editor.
- [ ] Required images are stored inside the assets folder.
- [ ] Required audio files are stored inside the assets folder.
- [ ] Required video files are stored inside the assets folder.
- [ ] No important file depends only on an online link.
- [ ] Offline ZIP file can be created.
- [ ] Offline ZIP file can be opened on another computer.
- [ ] Learners can complete tasks without internet if needed.

## Testing Checklist

Before final submission, test the project:

- [ ] Open README.md locally.
- [ ] Open all files in VS Code.
- [ ] Check all folder names.
- [ ] Check all file names.
- [ ] Check all links.
- [ ] Check online repository access.
- [ ] Download the project as ZIP.
- [ ] Open the ZIP offline.
- [ ] Confirm that learning materials are usable offline.
