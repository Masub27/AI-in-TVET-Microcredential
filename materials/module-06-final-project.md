# Module 6: Final Project

## Module Overview

This module guides learners to complete the final project for the AI-in-TVET Microcredential.

The final project is a complete AI-supported online/offline teaching workflow package. It should show that the learner can use GitHub, VS Code, AI support, LiaScript, and the BMAD Method to create structured educational content.

## Module Aim

The aim of this module is to help learners combine all previous modules into one final micro-credential project.

## Learning Outcomes

After completing this module, learners will be able to:

1. Plan a complete AI-supported teaching workflow project.
2. Organize course documentation, modules, tasks, and templates.
3. Apply the BMAD Method to structure the project.
4. Use AI support to create and improve learning materials.
5. Prepare materials for online and offline learning.
6. Submit final evidence for micro-credential completion.

## Final Project Description

Learners must create a complete teaching workflow package for a small TVET or digital education topic.

The final project should demonstrate:

- Clear project structure
- Clear learning outcomes
- Practical learning materials
- AI-supported content creation
- BMAD-based planning
- Online and offline delivery
- Assessment strategy
- Certificate or badge template

## Required Final Project Structure

The final project should include:

- README.md
- docs/course-overview.md
- docs/learning-outcomes.md
- docs/assessment-strategy.md
- docs/microcredential-definition.md
- docs/eu-standard-elements.md
- materials/
- tasks/
- templates/
- checklists/
- Outputs/

## Required Content

The final project must include:

1. Project title
2. Project overview
3. Target group
4. Learning outcomes
5. Course modules
6. Learner tasks
7. Assessment strategy
8. Quality checklist
9. Certificate or badge template
10. Online/offline delivery explanation

## Final Project Steps

### Step 1: Select a Topic

Choose one topic related to TVET, AI, or digital teaching.

Example topics:

- AI-supported lesson planning
- GitHub for teaching materials
- Online and offline learning design
- LiaScript for interactive teaching
- Digital assessment in TVET
- BMAD-based course creation

### Step 2: Define the Target Group

Clearly describe who the project is for.

Examples:

- TVET students
- Teachers
- Trainers
- Workshop participants
- Educational technology beginners

### Step 3: Write Learning Outcomes

Write clear learning outcomes.

Each outcome should describe what learners will be able to do after completing the micro-credential.

### Step 4: Create Learning Materials

Create at least two learning modules.

Each module should include:

- Module overview
- Aim
- Learning outcomes
- Key concepts
- Learning activity
- Task
- Evidence to submit
- Assessment criteria
- Completion checklist

### Step 5: Create Learner Tasks

Create at least two learner tasks inside the tasks folder.

Each task should include:

- Task title
- Instructions
- Expected output
- Evidence to submit
- Assessment criteria

### Step 6: Create Assessment Strategy

Create or update the assessment strategy.

The assessment should include:

- Module tasks
- Final project
- Evidence submission
- Passing requirement
- Feedback process

### Step 7: Create Certificate or Badge Template

Create a certificate or badge template.

The certificate should include:

- Learner name
- Title of micro-credential
- Awarding body
- Date of issuing
- Learning outcomes
- Assessment result
- Signature or instructor name

### Step 8: Prepare Online and Offline Versions

Prepare the project so it can be used:

- Online through GitHub, GitHub Pages, or LiaScript
- Offline through a downloaded folder or ZIP package

### Step 9: Review the Project

Use the quality checklist before submission.

Check:

- Are all required files included?
- Are learning outcomes clear?
- Are tasks practical?
- Is assessment transparent?
- Can the project work online?
- Can the project work offline?
- Is the folder structure clear?

### Step 10: Submit Final Evidence

Submit:

- GitHub repository link
- Screenshot of VS Code project folder
- Screenshot of GitHub repository
- Offline ZIP file if required
- Short reflection on the project

## Final Reflection

Write a reflection of 200–300 words.

Answer these questions:

1. What did you create?
2. How did you use the BMAD Method?
3. How did AI support your content creation?
4. What did you improve manually?
5. How can this project support TVET learning?
6. How can the project be used online and offline?

## Final Project Assessment Criteria

| Criteria | Description | Percentage |
|---|---|---:|
| Project Structure | Folder and files are complete and organized. | 20% |
| Learning Materials | Modules are clear, useful, and connected to outcomes. | 20% |
| BMAD Application | BMAD Method is visible in planning and structure. | 15% |
| AI Use and Improvement | AI support is used responsibly and improved manually. | 15% |
| Online/Offline Delivery | Project can be used online and offline. | 15% |
| Assessment and Reflection | Assessment strategy and reflection are complete. | 15% |

## Passing Requirement

Learners must achieve at least 60% overall and submit the final project evidence.

## Evidence to Submit

Final evidence should include:

- GitHub repository link
- Completed project folder
- Final reflection
- Screenshots
- Certificate or badge template
- Offline package if required
