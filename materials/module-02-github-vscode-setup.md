# Module 2: GitHub and VS Code Setup

## Module Overview

This module introduces learners to the basic setup of GitHub and VS Code for creating and managing educational content.

GitHub is used to store, share, and publish project files. VS Code is used to write and edit learning materials, tasks, templates, and documentation.

## Module Aim

The aim of this module is to help learners set up a simple GitHub and VS Code workflow for creating AI-supported teaching materials.

## Learning Outcomes

After completing this module, learners will be able to:

1. Explain the purpose of GitHub in educational content development.
2. Explain the purpose of VS Code in a teaching workflow.
3. Create a structured project folder.
4. Edit Markdown files in VS Code.
5. Save and organize learning materials.
6. Prepare files for uploading to GitHub.

## Key Concepts

### GitHub

GitHub is an online platform for storing and managing project files. In this micro-credential, GitHub is used to:

- Store course materials.
- Share learning resources.
- Track project progress.
- Publish online learning content.
- Submit final project evidence.

### VS Code

VS Code is a code and text editor. In this micro-credential, VS Code is used to:

- Open the project folder.
- Edit Markdown files.
- Organize teaching materials.
- Create tasks and templates.
- Manage the project structure.

### Markdown

Markdown is a simple writing format used for documentation and learning materials.

Markdown uses simple symbols, for example:

- Use one hashtag for a main heading.
- Use two hashtags for a subheading.
- Use hyphens for bullet points.
- Use numbers for ordered lists.

Markdown is useful because it is simple, readable, and works well with GitHub.

## Required Tools

Learners need:

- A computer or laptop.
- Internet connection for GitHub setup.
- VS Code installed.
- GitHub account.
- Git installed on the computer.

## Step 1: Create a GitHub Account

Go to GitHub and create an account.

Use a professional username if possible.

## Step 2: Create a New Repository

Create a new GitHub repository with a clear project name.

Example repository name:

AI-in-TVET-Microcredential

Recommended settings:

- Visibility: Public or Private
- Add README: Yes
- License: Optional

## Step 3: Open Project in VS Code

Open VS Code and select:

File → Open Folder

Choose the project folder.

## Step 4: Create the Project Structure

The project should include folders such as:

- assets/
- checklists/
- data/
- docs/
- examples/
- materials/
- Outputs/
- skeletons/
- tasks/
- templates/
- README.md

## Step 5: Edit README.md

The README file explains the whole project.

It should include:

- Project title
- Project overview
- Aim
- Target group
- Tools
- Learning outcomes
- Project structure
- Assessment
- Final output

## Step 6: Save Changes

After editing any file in VS Code, save it with:

Command + S

## Step 7: Prepare for GitHub Upload

After creating and editing files, learners can upload the project to GitHub.

Basic Git commands:

git init

git add .

git commit -m "Initial project setup"

git branch -M main

git remote add origin YOUR-GITHUB-REPOSITORY-LINK

git push -u origin main

Replace YOUR-GITHUB-REPOSITORY-LINK with the real GitHub repository link.
