# Module 5: Online and Offline Course Package

## Module Overview

This module explains how to prepare teaching materials for both online and offline learning.

Online learning means learners can access materials through GitHub, GitHub Pages, or LiaScript. Offline learning means learners can download the project folder and use the materials without internet.

## Module Aim

The aim of this module is to help learners create a teaching package that can be used online and offline.

## Learning Outcomes

After completing this module, learners will be able to:

1. Explain the difference between online and offline course delivery.
2. Organize learning materials for online access.
3. Organize learning materials for offline use.
4. Prepare a downloadable course package.
5. Use GitHub for sharing and publishing learning materials.
6. Check whether the course package is complete and usable.

## Online Course Package

An online course package can be shared through:

- GitHub repository
- GitHub Pages
- LiaScript course link
- Online documentation
- Shared learning platform

Online learning is useful because:

- Learners can access materials from different locations.
- Teachers can update content quickly.
- GitHub can track changes.
- Materials can be shared through a link.
- Learners can submit evidence digitally.

## Offline Course Package

An offline course package can be used without internet.

An offline package can include:

- Markdown files
- PDF files
- HTML files
- Images
- Videos
- Audio files
- Tasks
- Templates
- Assessment rubrics
- Certificate templates

Offline learning is useful because:

- Learners can study without internet.
- Materials can be shared through USB or ZIP file.
- The course can be used in low-connectivity areas.
- Learners can keep a local copy of the materials.

## Recommended Folder Structure

The course package should follow this structure:

- assets/
- checklists/
- data/
- docs/
- examples/
- materials/
- Outputs/
- skeletons/
- tasks/
- templates/
- README.md

## Required Files for Online and Offline Use

The minimum course package should include:

1. README.md
2. docs/course-overview.md
3. docs/learning-outcomes.md
4. docs/assessment-strategy.md
5. materials/module files
6. tasks/task files
7. templates/certificate-template.md
8. checklists/quality checklist

## Step 1: Prepare the Online Version

For the online version, learners should:

1. Create or update a GitHub repository.
2. Upload all project files.
3. Check that README.md is clear.
4. Make sure folder names are understandable.
5. Add links to modules and tasks.
6. Use GitHub Pages or LiaScript if required.

## Step 2: Prepare the Offline Version

For the offline version, learners should:

1. Keep all required files inside the project folder.
2. Make sure files can open locally.
3. Add images and media inside the assets folder.
4. Avoid broken links.
5. Create a ZIP file of the project folder.
6. Test the folder without internet.

## Step 3: Test the Course Package

Before submission, learners should check:

- Can the README be opened?
- Are all modules included?
- Are tasks included?
- Are assessment criteria included?
- Are templates included?
- Are images and media stored correctly?
- Can the folder be used without internet?
- Can the GitHub version be accessed online?

## Learning Activity

Prepare a small online/offline course package using the project structure.

The package should include:

- One course overview
- Two learning modules
- Two learner tasks
- One assessment strategy
- One certificate or badge template

## Task

Create an online/offline course package for a small AI-in-TVET learning topic.

Your package must include:

1. Project title
2. Course overview
3. Learning outcomes
4. At least two modules
5. At least two tasks
6. Assessment strategy
7. Certificate or badge template
8. Online or offline delivery explanation

## Evidence to Submit

Submit one or more of the following:

- GitHub repository link
- Screenshot of GitHub repository
- Screenshot of VS Code folder
- ZIP file of offline package
- Screenshot showing local offline access

## Assessment Criteria

| Criteria | Description |
|---|---|
| Completeness | Required files and folders are included. |
| Online Access | The project can be shared or viewed online. |
| Offline Access | The project can be opened and used offline. |
| Organization | Folder structure is clear and logical. |
| Usability | Learners can understand and use the materials. |
| Evidence | Screenshots, links, or ZIP file are provided. |

## Short Summary

Online and offline course packages help make learning more flexible. GitHub and LiaScript can support online access, while a well-organized folder or ZIP file can support offline learning. In this module, learners prepare a teaching package that can work in both formats.
