# Task 1: Create a GitHub Repository

## Task Overview

In this task, learners create a GitHub repository for the AI-in-TVET Microcredential project.

The repository will be used to store, organize, and share all project files.

## Aim

The aim of this task is to help learners create a professional GitHub repository for educational content development.

## Instructions

Follow these steps:

1. Open GitHub in your browser.
2. Sign in to your GitHub account.
3. Click **New Repository**.
4. Write a clear repository name.
5. Add a short description.
6. Choose public or private visibility.
7. Create the repository.
8. Open the repository page.
9. Copy the repository link.

## Suggested Repository Name

Use this name:

AI-in-TVET-Microcredential

## Suggested Description

AI-supported microcredential project for online and offline teaching workflow creation in TVET.

## Repository Settings

Recommended settings:

- Repository name: AI-in-TVET-Microcredential
- Visibility: Public or Private
- README: Yes, if creating directly on GitHub
- License: Optional
- Gitignore: Optional

## Expected Output

At the end of this task, the learner should have:

- A GitHub repository
- A repository link
- A README file or project folder ready for upload

## Evidence to Submit

Submit one of the following:

- GitHub repository link
- Screenshot of the repository page
- Screenshot showing repository name and files

## Assessment Criteria

| Criteria | Description |
|---|---|
| Repository Created | A GitHub repository is created successfully. |
| Clear Name | The repository has a clear and relevant name. |
| Description | The repository includes a short project description. |
| Access | The repository can be opened or viewed by the reviewer. |
| Evidence | Link or screenshot is submitted. |

This task creates the GitHub space for the AI-in-TVET Microcredential project. The repository will be used to store and share all learning materials, tasks, templates, and final project evidence.
