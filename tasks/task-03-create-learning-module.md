# Task 3: Create a Learning Module

## Task Overview

In this task, learners create one learning module for the AI-in-TVET Microcredential project.

The module should be written in Markdown and saved inside the `materials/` folder.

## Aim

The aim of this task is to help learners design a clear and structured learning module for online and offline use.

## Instructions

Follow these steps:

1. Open the project folder in VS Code.
2. Go to the `materials/` folder.
3. Create a new Markdown file.
4. Give the file a clear name.
5. Write the module title.
6. Add a module overview.
7. Add the module aim.
8. Write learning outcomes.
9. Add key concepts.
10. Add a learning activity.
11. Add a learner task.
12. Add evidence to submit.
13. Add assessment criteria.
14. Add a completion checklist.
15. Save the file.

## Suggested File Name

Use a clear file name, for example:

`module-ai-supported-lesson-planning.md`

## Required Module Sections

The module should include:

1. Module title
2. Module overview
3. Module aim
4. Learning outcomes
5. Key concepts
6. Learning activity
7. Task
8. Evidence to submit
9. Assessment criteria
10. Completion checklist
11. Short summary

## Example Module Topic

Example topic:

**AI-Supported Lesson Planning in TVET**

Possible learning outcomes:

1. Explain how AI can support lesson planning.
2. Create one lesson activity using AI support.
3. Review and improve AI-generated content.
4. Save the final learning activity as a Markdown file.

## Expected Output

At the end of this task, the learner should have:

- One complete learning module.
- The module saved as a Markdown file.
- The module placed inside the `materials/` folder.

## Evidence to Submit

Submit one of the following:

- Markdown file of the learning module.
- Screenshot of the module in VS Code.
- GitHub link to the module file.

## Assessment Criteria

| Criteria | Description |
|---|---|
| Module Structure | The module includes all required sections. |
| Learning Outcomes | Outcomes are clear and connected to the topic. |
| Learning Activity | Activity is practical and suitable for the target group. |
| Task | Learner task is clear and achievable. |
| Assessment Criteria | Criteria explain how the task will be checked. |
| Markdown Use | File uses clear Markdown headings and lists. |

## Completion Checklist

Before submitting, check that:

- [ ] Module file is created.
- [ ] Module title is clear.
- [ ] Overview is included.
- [ ] Aim is included.
- [ ] Learning outcomes are included.
- [ ] Activity and task are included.
- [ ] Assessment criteria are included.
- [ ] File is saved in the `materials/` folder.
- [ ] Evidence is ready for submission.

## Short Summary

This task helps learners practise creating a structured learning module. The module should be clear, practical, and suitable for online and offline learning.
