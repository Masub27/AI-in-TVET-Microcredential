# Assessment Strategy

## Purpose
This document explains how learners will be assessed in the AI-in-TVET Microcredential.

The assessment is practical and evidence-based. Learners must complete module tasks and submit a final online/offline teaching workflow package.

## Assessment Overview

| Assessment Component | Description | Percentage |
|---|---|---:|
| Module Tasks | Learners complete practical tasks in each module. | 30% |
| Online/Offline Teaching Package | Learners prepare learning materials for online and offline use. | 30% |
| Final Project | Learners submit a complete AI-supported teaching workflow project. | 40% |

## Passing Requirement

To successfully complete the micro-credential, learners must achieve at least:

**60% overall**

Learners must also submit the final project evidence.

## Assessment Evidence

Learners must provide evidence such as:

1. GitHub repository link or offline project folder.
2. Completed Markdown learning materials.
3. Screenshots of VS Code and GitHub workflow.
4. Completed learner tasks.
5. Final teaching workflow package.
6. Short reflection on the use of AI and the BMAD Method.

## Module Task Assessment

Each module task will be checked using the following criteria:

| Criteria | Description |
|---|---|
| Completion | The task is completed and submitted. |
| Clarity | The work is clear and understandable. |
| Relevance | The work is connected to AI-supported teaching workflow creation. |
| Evidence | The learner provides files, screenshots, or links as proof. |

## Final Project Assessment Rubric

| Criteria | Excellent | Good | Needs Improvement |
|---|---|---|---|
| Project Structure | Folder structure is complete and well organized. | Folder structure is mostly complete. | Folder structure is incomplete or unclear. |
| Learning Materials | Materials are clear, useful, and well designed. | Materials are understandable but need small improvements. | Materials are weak or incomplete. |
| Use of Tools | GitHub, VS Code, AI support, LiaScript, and BMAD are used correctly. | Most tools are used correctly. | Tools are not used clearly or correctly. |
| Online/Offline Delivery | Project works for both online and offline use. | Project mostly supports online/offline use. | Online/offline use is unclear. |
| Assessment Design | Tasks and assessment criteria are transparent. | Assessment is included but needs improvement. | Assessment is missing or unclear. |
| Reflection | Reflection clearly explains learning and improvement. | Reflection is present but general. | Reflection is missing or too short. |

## Grading Scale

| Score | Result |
|---:|---|
| 90–100% | Excellent |
| 75–89% | Good |
| 60–74% | Passed |
| Below 60% | Not passed |

## Feedback Process

Learners will receive feedback on:

- Quality of learning materials.
- Correct use of GitHub and VS Code.
- Application of the BMAD Method.
- Online/offline usability.
- Final project completeness.

## Quality Assurance

Assessment quality will be supported through:

1. Clear learning outcomes.
2. Transparent assessment criteria.
3. Practical evidence submission.
4. Checklists for review.
5. Learner feedback.
6. Instructor or peer review.

## Final Completion

A learner completes the micro-credential when all required tasks are submitted, the final project is accepted, and the minimum passing requirement is achieved.
