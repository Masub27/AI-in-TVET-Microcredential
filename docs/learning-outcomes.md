# Learning Outcomes

## Purpose
This document defines the learning outcomes of the AI-in-TVET Microcredential.

Learning outcomes describe what learners should know, understand, and be able to do after completing the micro-credential.

## Main Learning Outcomes

After completing this micro-credential, learners will be able to:

1. Explain the concept and purpose of micro-credentials in education and TVET.
2. Describe the basic European and UNESCO principles of micro-credentials.
3. Set up a GitHub repository for managing educational content.
4. Use VS Code to create and edit Markdown-based learning materials.
5. Apply the BMAD Method to structure an educational content creation workflow.
6. Use GitHub Copilot or AI support to generate and improve teaching materials.
7. Create online learning content using GitHub and LiaScript.
8. Prepare offline learning materials that can be downloaded and used without internet.
9. Design practical learner tasks and assessment activities.
10. Create a final AI-supported online/offline teaching workflow package.

## Knowledge Outcomes

Learners will understand:

- What micro-credentials are.
- Why micro-credentials are useful for flexible learning.
- How AI can support teaching workflow creation.
- How GitHub and VS Code support educational content development.
- How online and offline learning materials can be organized.

## Skill Outcomes

Learners will be able to:

- Create a structured course folder.
- Write learning materials in Markdown.
- Use GitHub for storing and sharing teaching content.
- Use VS Code for editing and organizing project files.
- Create learner tasks and assessment documents.
- Prepare a certificate or badge template.
- Present a final project as evidence of learning.

## Competence Outcomes

Learners will be able to independently design a small AI-supported teaching workflow for TVET and make it available for online and offline learning.

## Evidence of Achievement

Learners will demonstrate achievement through:

1. Completed module tasks.
2. GitHub repository or offline project folder.
3. Screenshots of the workflow.
4. Final teaching workflow package.
5. Reflection on the use of AI and BMAD Method.
