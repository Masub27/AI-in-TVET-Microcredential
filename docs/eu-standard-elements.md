# EU Standard Elements for Micro-Credentials

## Purpose of This Document
This document explains how the AI-in-TVET Microcredential follows the European Union approach to micro-credentials.

The European approach provides common standards so that micro-credentials can be transparent, comparable, recognised, and portable across education and work contexts.

## EU Definition of Micro-Credential
According to the European approach, a micro-credential is a record of the learning outcomes that a learner has acquired after a small volume of learning.

These learning outcomes must be assessed against transparent and clearly defined standards.

## Mandatory EU Elements

This micro-credential should include the following required elements:

| EU Element | Application in This Project |
|---|---|
| Identification of the learner | Learner name will be included in the certificate or badge. |
| Title of the micro-credential | AI-in-TVET Microcredential |
| Country or region of the issuer | Germany |
| Awarding body | Otto-von-Guericke University Magdeburg / Project context |
| Date of issuing | Added when the learner completes the micro-credential |
| Learning outcomes | Listed in the README and learning outcomes document |
| Notional workload | Estimated learning time will be included in the course overview |
| Level of learning experience | Beginner to intermediate level |
| Type of assessment | Practical tasks, final project, and evidence submission |
| Form of participation | Online and offline learning |
| Quality assurance | Checklists, assessment rubric, and review process |

## Optional Elements

Where relevant, this project may also include:

- Prerequisites for participation.
- Supervision or identity verification during assessment.
- Grade or pass/fail result.
- Stackability with other micro-credentials.
- Further information through GitHub documentation.

## Quality Principle
The micro-credential should be clearly documented and reviewed. Quality assurance will be supported through checklists, assessment rubrics, learner feedback, and instructor review.

## Transparency Principle
The learning outcomes, workload, assessment type, tools, and final output must be clearly explained so learners and teachers understand the value of the micro-credential.

## Relevance Principle
The micro-credential is relevant because it focuses on AI-supported teaching workflow creation in TVET. It connects educational technology, GitHub, VS Code, GitHub Copilot, LiaScript, and the BMAD Method.

## Valid Assessment Principle
Learners must complete practical tasks and submit evidence. The final project will be assessed using transparent criteria.

## Portability Principle
The learner should be able to share the final certificate, badge, or GitHub portfolio as evidence of achievement.

## Learner-Centred Principle
The micro-credential is designed for learners who want flexible online and offline learning. It supports self-directed learning and practical project work.

## Application in This Project
This project applies the EU approach by including:

1. Clear learning outcomes.
2. A defined title.
3. Practical assessment.
4. Online and offline participation.
5. Workload information.
6. Quality assurance checklists.
7. Certificate or badge template.
8. Final project evidence.
